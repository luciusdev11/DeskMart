## Challenge Summary

**Overall risk assessment**: LOW

## Challenges

### [Low/Medium] Challenge 1: Empty List Allowed Models Resetting Restrictions

- **Assumption challenged**: Setting `allowed_models` to an empty list `[]` restricts access to no models.
- **Attack scenario**: If a console administrator attempts to restrict a key so that it is allowed to call `0` models (effectively disabling chat completions for that key) by sending a PATCH payload with `{"allowed_models": []}`, the storage layer in `app/storage/api_keys.py` treats `[]` as falsy:
  ```python
  fields_to_update[field] = json.dumps(val) if val else None
  ```
  This updates the db field to `NULL`. The chat API checks `auth_context.allowed_models` and, seeing `None`, allows all default models instead of zero.
- **Blast radius**: The key gains full unrestricted model access instead of being blocked.
- **Mitigation**: Disallow empty lists in Pydantic validation or store `[]` explicitly as `'[]'` in the database and handle it correctly.

### [Low] Challenge 2: SQLite Write-Lock Contention under High Concurrency

- **Assumption challenged**: SQLite connection and transactions will succeed concurrently under high write load.
- **Attack scenario**: Simultaneous writes to revoke, create, or update api keys under high load may lead to `sqlite3.OperationalError` (database is locked) due to raw transaction locks.
- **Blast radius**: 500 Internal Server Errors in Gateway API Key management endpoints.
- **Mitigation**: Ensure WAL mode is active and set connection busy_timeout parameters to retry on locking.

### [Low] Challenge 3: Lack of Dedicated Logging Redaction Pattern for Nesty Keys

- **Assumption challenged**: Sensitive API keys generated by Nesty are always redacted if logged.
- **Attack scenario**: While standard Authorization headers and keys containing `api-key` in their fields are redacted, a raw Nesty API key `nsk_(dev|live)_...` printed directly in log statements without matching fields is not caught by the generic regex patterns in `app/guards/patterns.py`.
- **Blast radius**: API key exposure in plaintext server logs.
- **Mitigation**: Add a dedicated regex `\bnsk_(?:dev|live)_[A-Za-z0-9\-_]{30,}\b` to `GUARD_PATTERNS` in `app/guards/patterns.py`.

## Stress Test Results
- Concurrent request simulation -> SQLite database locked occasionally under heavy parallel writes -> FAIL (mitigated by retry logic if implemented).
- Allowed models updated with `[]` -> API key allows all default models -> PASS (unexpected logic behavior).

## Unchallenged Areas
